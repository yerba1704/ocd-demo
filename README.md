# Introduction

This repository demonstrates the strengths of __ora* CODEDOC__.
For more information see [ocd](https://github.com/yerba1704/ocd).

## Installation

By executing the script `admin_install.sql`, a new user called `OCD_DEMO` is created along with some sample packages.

The `OCD_DEMO` user receives the following privileges:

- create session
- create procedure

## Sample packages

The following sample packages are included in `OCD_DEMO` after installation:

- [PKG_SAMPLE](ddl/package/pkg_sample.pks) is the official sample package, which is fully documented using the __ora* CODEDOC syntax (OCDs)__. Read [here](https://github.com/yerba1704/ocd/blob/main/doc/ocds.md) for more information.

- [APEX_CSS](ddl/package/apex_css.pks) is the original package from Oracle APEX and can be used for comparison.

- [APEX_CSS_MODIFIED](ddl/package/apex_css_modified.pks) is the original Oracle APEX package, reformatted in the style of __ora* CODEDOC syntax (OCDs)__.

## Sample publisher

The [hi/](hi/) folder contains some scripts for building a sample documentation:

- [code.sql](hi/oraclehtml/code.sql) generates a simple website similar to Oracle’s [APEX API](https://docs.oracle.com/en/database/oracle/apex/24.2/aeapi/index.html). For the package `APEX_CSS`, you can compare the result generated by __ora* CODEDOC__ [[1](doc/apex_css.html)] with the official Oracle APEX documentation website [[2](https://docs.oracle.com/en/database/oracle/apex/24.2/aeapi/APEX_CSS.html)].

The [ai/](ai/) folder contains some websites made by popular AI websites with the JSON provided by __ora* CODEDOC__.

## Requirements

**ora* CODEDOC** will run on any Oracle Database version 18c or above.
